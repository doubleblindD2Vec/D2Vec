In this experiment - API tutorial fragment search (code-to-text retrieval), our goal is to retrieve the relevant tutorial fragment given an API query. Specifcally, we used D2VEC to compute the relevance score sim(A,F) which is cosine similarity between the vector VA of the API A after training and the vector VF for the fragment F.

Data Collection

For a comparative study, we chose to use the state-of-the-art tool FRAFT2 where the authors also kindly provided the tool and dataset with the ground truth used in their experimental studies.
McGill Tutorial Oracle: Download
To train the models to build vector representations, we used Eclipse JDT compiler to parse the source code including Javadoc of the five libraries (Jodatime, Apache Common Math3, Collection Official, Jenkov Collection, Smack).
Documentation of 5 libraries: Download

Result

D2VEC achieved precision, recall, and F-score of 73.2%, 90.5%, and 80.5%, respectively. Comparing with Ye et al1, D2VEC had much better precision (73.2% versus 62.5%), while maintaining similar recall. Thus, D2VEC resulted in an improvement of nearly 8% in F-score. It also achieved slightly better results than FRAFT. In particular, D2VEC increased recall by 4.5% while loosing only 1% precision, leading to an overall increasing F-score of 2%.

[1] X. Ye, H. Shen, X. Ma, R. Bunescu, and C. Liu. From word embeddings to document similarities for improved information retrieval in software engineering. In Proceedings of the 38th International Conference on Software Engineering, ICSE ’16, pages 404–415, New York, NY, USA, 2016. ACM.
[2] H. Jiang, J. Zhang, Z. Ren, and T. Zhang. An unsupervised approach for discovering relevant tutorial fragments for apis. In Proceedings of the 39th International Conference on Software Engineering, ICSE ’17, pages 38–48. IEEE Press, 2017.
