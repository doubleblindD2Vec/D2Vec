Capturing API Descriptions with Vector Representations to Improve Retrieval in Sofware Engineering Applications

  • Home
  • API code search
  • Mapping Between API Libraries
  • API tutorial fragment search

When being trained on documentation and tutorials, Word2Vec is useful in estimating the relevance between texts and code elements. However, existing Word2Vec-based approaches to measure document similarities aggregate the Word2Vec vectors of individual words or code elements to build the representation of a document as if the words in the document are independent. Thus, the semantics of API description or code fragments are not well represented.
In this work, we present D2VEC, a neural network model to represent an API with a distributed vector that captures the entire description of that API and the related APIs in an API documentation. The semantics are considered by connecting the current API topic under description to all the text phrases within the description of that API. D2Vec has the same origin properties of Word2Vec, but captures more the semantics of API description or code fragments with the API topic. We demonstrate the usefulness and good performance of D2VEC in three applications: API code search (text-to-code retrieval), API tutorial fragment search (code-to-text retrieval), and mining single API mappings between software libraries (code-to-code retrieval).
